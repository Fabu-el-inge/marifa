  ESTADO DEL PROYECTO
==============================

QUÉ ES MARIFÁ
-----------
App web para que una cantante gestione setlists.
- Armás un Show (fecha, venue, ciudad)
- Le asignás canciones desde el catálogo
- Reordenás con drag & drop
- Compartís link público con los músicos (sin que tengan que loguearse)
- Podés exportar PDF

STACK TÉCNICO
-------------
- Backend:   Python 3.11 + Flask 3.x
- Base de datos: SQLite (archivo: instance/marifa.db)
- ORM:       SQLAlchemy + Flask-Migrate
- Auth:      Flask-Login
- CSS:       Tailwind CSS + clases custom en app/static/css/marifa.css
- JS:        Alpine.js + SortableJS (drag & drop) + fetch
- PDF:       WeasyPrint

MODELOS DE BASE DE DATOS
------------------------
User:        id, username, email, password_hash, created_at
Song:        id, title, original_artist, genre, key, bpm, duration_sec,
             lyrics, musician_notes, is_active, created_at, updated_at
Show:        id, name, date, venue, city, general_notes,
             public_token, is_public, created_at, updated_at
SetlistItem: id, show_id, song_id (nullable), position, notes_override,
             is_break, break_label, created_at

RUTAS PRINCIPALES
-----------------
/auth/login              Login
/songs/                  Catálogo de canciones
/songs/new               Nueva canción
/songs/<id>/edit         Editar canción
/shows/                  Lista de shows
/shows/new               Nuevo show
/setlist/<id>/builder    Armador drag & drop
/p/<token>               Vista pública para músicos (sin login)

==============================
  PENDIENTE — DEPLOY
==============================

PASOS EN ORDEN
--------------
1. Subir código a GitHub
   - Crear repo en github.com (privado o público)
   - git init, git add ., git commit -m "Initial commit"
   - git remote add origin <url-del-repo>
   - git push

2. Migrar base de datos a Supabase (PostgreSQL)
   - Crear cuenta en supabase.com
   - Crear nuevo proyecto → copiar la "Connection string" (PostgreSQL)
   - En .env cambiar DATABASE_URL a la de Supabase
   - Ejecutar: flask db upgrade (crea las tablas en Supabase)
   - Ejecutar: python seed.py (opcional, para datos de prueba)
   - Volver a crear el usuario marifa en producción

3. Deployar en Railway
   - Crear cuenta en railway.app
   - Nuevo proyecto → conectar repositorio de GitHub
   - Agregar variables de entorno (las del .env)
   - Railway detecta Python automáticamente con gunicorn
   - La app queda online en una URL tipo marifa.up.railway.app

VARIABLES DE ENTORNO PARA PRODUCCIÓN (.env)
--------------------------------------------
SECRET_KEY=<clave-secreta-larga>
DATABASE_URL=postgresql://... (la de Supabase)
FLASK_ENV=production
